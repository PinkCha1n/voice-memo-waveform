# AI 编写过程提示词

以下是本次实现可以直接用于提交或二次修改的提示词记录。

## 1. 原始需求

写一个手机上能运行的组件，组件接收录音数据并根据音量数据展示一个组件，如图所示。高度在 12dp-58dp 之间，两格之间间距 6dp，根据时间从右往左向走，动画参考 iPhone 语音备忘录。

## 2. 技术落地

请用原生 Web Component 实现一个可在手机浏览器直接运行的波形组件，不依赖 React / Flutter / React Native / Vue。要求：

- 接收录音音量数据
- 波形从右向左滚动
- 条高 12-58px
- 条间距 6px
- 动画参考 iPhone 语音备忘录
- 给出一个可运行的 demo 页面

## 3. 细化实现

请把组件写成一个可复用自定义元素，支持如下接口：

- `pushLevel(level, timestamp?)`
- `setLevels(levels, startTime?)`
- `clear()`
- 支持属性：`min-height`、`max-height`、`gap`、`bar-width`、`sample-interval`、`history-ms`、`color`

## 4. 演示要求

请额外提供：

- 一个移动端友好的示例页面
- 一个本地文件上传入口
- 一个播放与暂停按钮
- 一个 README，说明如何运行和如何接入真实录音数据

## 5. 录屏建议

如果你要提交过程视频，可以按下面的顺序录：

1. 先展示提示词记录
2. 再展示组件源码
3. 再打开 demo 演示录音文件添加
4. 点击播放和暂停，演示波形运动与冻结
5. 最后切到 README 说明交付方式

## 6. 可直接复用的二次提示词

把波形做成更像 iPhone 语音备忘录的效果：背景更克制，条形更顺滑，播放时从右向左运动，暂停时冻结，并且支持从本地录音文件中读取真实音量数据。

## 7. 本次改版提示词

请把现有模拟波形改为真实录音文件版，使用浏览器本地上传的音频文件作为输入，删除模拟按钮和假数据输入，改成可以选择录音文件并解析真实音量数据。条高限制在 12px 到 58px，条间距保持 6px，新增播放和暂停按钮，播放时波形运动，暂停时波形停止，输出一个可以直接在手机浏览器运行的示例页面。
